# Contributing

Спасибо за интерес к Workbench! Любые улучшения — баг-фиксы, новые возможности фреймворка, документация — приветствуются.

## С чего начать

1. Поднимите проект локально (см. [README](README.md#быстрый-старт)).
2. Создайте ветку от `main`: `git checkout -b feat/short-description`.
3. Внесите изменения, добавьте тесты, убедитесь, что всё зелёное.
4. Откройте Pull Request с понятным описанием.

## Перед PR

```bash
./vendor/bin/pint        # автоформат (Laravel Pint)
php artisan test         # тесты должны проходить
npm run build            # фронтенд должен собираться
```

## Стиль кода

- **PHP** — стандарт Laravel, форматирование через Pint (`./vendor/bin/pint`).
- **Vue/JS** — следуйте стилю существующих компонентов (`<script setup>`, composables в `resources/js/composables`).
- Комментарии и UI-строки в проекте на русском — придерживайтесь того же языка в затрагиваемых местах.

## Добавление сервлета

Если вы добавляете пример сервлета в основной репозиторий — он должен быть **безопасным и без учётных данных**: только публичные источники, без реальных логинов/паролей/токенов. Сервлеты под конкретные сайты с авторизацией держите в своём форке.

## Чего не делать

- Не коммитьте `.env`, ключи, токены, реальные учётные данные или дампы данных.
- Не добавляйте бинарные артефакты сборки (`public/build`, `node_modules`, `vendor`).

## Сообщить об уязвимости

Вопросы безопасности — не через публичные issue, а на e-mail сопровождающего, указанный в профиле репозитория.
